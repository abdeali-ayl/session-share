This file contains details of the revenue calculation every month. We have a dump of order transactions data that gets condensed into brand - locations, total GMV, and our monthly commission structure is typically a minimum guarantee or a revenue share, whichever is higher at a brand - location level. That revenue could be based on gross merchandise value (GMV), which is the total brand sales, or net merchandise value (NMV), which is GMV minus discounts. The GMV in our data set is equal to subtotal plus charges. That's the column names, and charges refers to packaging charges and subtotal refers to the total amount of value in the cart of the customer.

Every month will have different tabs. This is typically because analysis is conducted on these tabs. You will find one tab which has a summary of GMV, Kytchens commission, which is also the same as the revenue. In most cases, you will also see a total at the bottom. That total is a very important number because that is what actually is getting reported, or that's what we test against to see whether our data capture is correct or not.

Remember that this is financial data, so accuracy is of utmost importance.

The approach that I would take if I had to manually do this would be to:

  1. Open each file.
  2. Understand what the tab structure looks like.
  3. Find the summary tab.
  4. Restructure and put it into one new sheet in one tab.
  5. Start appending below that for every single month in the same structure

I will then be able to create a pivot table in the fresh sheet on one dataset, which is a summary of all summaries present in the individual sheets for the month.

I would then run tests to verify the total for every month against the sheet of every month.

Phase 3 — Load each month (chronological)

  1. Read full summary tab with UNFORMATTED_VALUE (avoids "15,000" formatted strings)
  2. Identify header row (scan first 3 rows — some tabs have a title row before headers)
  3. Identify Grand Total row (last data row where Brand/Location blank but GMV/Commission numeric) — save for verification; exclude from data rows
  4. Normalize column names using mapping table above
  5. Fill down blank Brand cells (merged cells show brand name only in first row of group)
  6. Strip trailing/leading spaces from Brand and Location
  7. Add Month column as first column
  8. Drop non-canonical columns
  9. Append to All Months Data tab
  10. Verify (see Phase 4)

Phase 4 — Verification (per month, immediately after loading)

  1. Sum GMV and Commission in Python for rows matching that month
  2. Compare against Grand Total row captured from source sheet
  3. Tolerance: ≤ ₹1 difference (floating point rounding)
  4. Write result to Verification tab: PASS or FAIL with expected vs actual — STOP and report before loading next month on FAIL

Do NOT use the pre-existing "Monthly Summary" sheet as the verification reference. The source of truth is the Grand Total row within each month's own source sheet tab.

Phase 5 — Format consolidated sheet
